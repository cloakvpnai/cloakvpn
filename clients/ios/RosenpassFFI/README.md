# RosenpassFFI

Swift FFI wrapper around the upstream `rosenpass` Rust crate, used by the
Cloak iOS app to run a post-quantum (Classic McEliece + ML-KEM) key
exchange in the **main app process**, then push the derived 32-byte PSK to
the Packet Tunnel Provider extension via App Group + `sendProviderMessage`.

## Why a separate crate

The upstream `rosenpass` crate is structured as a daemon binary plus a
library. Its public API is large and not yet stable for embedding. This
crate gives the iOS app a small, frozen Swift-facing surface (~5 calls)
that we control, so future churn in upstream rosenpass internals doesn't
ripple into Swift.

## Architecture

This crate runs ONLY in the main app, never in the NetworkExtension.
Reason: NEPacketTunnelProvider is capped at 50 MiB on iOS 15+, and one
McEliece-460896 public key alone is 524 KB; key generation peaks at
2-4 MB working set. The main app has no such cap, so all PQC work
happens there. The 32-byte PSK is the only crypto material that crosses
into the NE.

See `cloak-vpn/docs/IOS_PQC.md` for the full architectural rationale and
the day-1 cross-compile verification log.

## Building

### Prerequisites

```bash
brew install cmake rustup
rustup-init -y --default-toolchain stable
rustup toolchain install 1.88.0
rustup target add aarch64-apple-ios aarch64-apple-ios-sim --toolchain 1.88.0
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept
```

The 1.88.0 toolchain is the sweet spot. 1.95+ breaks `memchr`'s NEON
intrinsics on iOS-arm64; 1.77 (which upstream rosenpass pins) is too old
for `serde_spanned 1.1.1` from uniffi's dep graph (`edition2024`).

### Cross-compile for device + Simulator

```bash
# Device (iPhone, iPad)
cargo +1.88.0 build --release --target aarch64-apple-ios

# Apple Silicon Mac Simulator
SDK=$(xcrun --sdk iphonesimulator --show-sdk-path)
BINDGEN_EXTRA_CLANG_ARGS_aarch64_apple_ios_sim="--target=arm64-apple-ios14.0-simulator --sysroot=$SDK" \
    cargo +1.88.0 build --release --target aarch64-apple-ios-sim
```

Output: `target/<triple>/release/librosenpassffi.a`

### Build the XCFramework

```bash
xcodebuild -create-xcframework \
    -library target/aarch64-apple-ios/release/librosenpassffi.a \
    -headers include/ \
    -library target/aarch64-apple-ios-sim/release/librosenpassffi.a \
    -headers include/ \
    -output RosenpassFFI.xcframework
```

The Swift bindings file (`RosenpassFFI.swift`) is generated by `uniffi-bindgen`:

```bash
cargo run --bin uniffi-bindgen generate --library target/aarch64-apple-ios/release/librosenpassffi.a --language swift --out-dir Generated/
```

Drag both `RosenpassFFI.xcframework` and `Generated/RosenpassFFI.swift`
into the CloakVPN Xcode project.

## Swift usage (sketch)

```swift
import RosenpassFFI

// First install only (peaks ~2-4 MB; do off the main thread)
let kp = try generateStaticKeypair()
try Keychain.save(kp.secretKey, key: "rosenpass.secret")
try Keychain.save(kp.publicKey, key: "rosenpass.public")

// Per-session
let session = try RosenpassSession(
    ourSecretKey: try Keychain.load(key: "rosenpass.secret"),
    ourPublicKey: try Keychain.load(key: "rosenpass.public"),
    peerPublicKey: serverRosenpassPublicKey
)

// Drive the protocol
while running {
    switch try session.step() {
    case .sendMessage(let bytes):
        udpSocket.send(bytes, to: rosenpassEndpoint)
    case .derivedPsk(let psk):
        try tunnelProvider.session.sendProviderMessage(psk) { _ in }
    case .idle:
        try await Task.sleep(seconds: 120)
    }
}
```
